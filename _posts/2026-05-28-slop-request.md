*(noun, software engineering, informal)*

1. A pull request containing an excessive volume of low-value, AI-generated, poorly reviewed, or unnecessarily broad code changes, often obscuring the meaningful modifications under large amounts of incidental churn.

2. By extension, any code review submission whose size and noise level are disproportionate to the actual feature, fix, or refactor being proposed.

## Etymology

Blend of *slop* (internet slang for low-quality mass-generated AI content) and *pull request*.

## Usage examples

- “This was a two-line fix turned into a full slop request.”
- “Can you split this slop request into smaller commits before review?”
- “The actual bugfix is buried under 3,000 lines of slop request.”
